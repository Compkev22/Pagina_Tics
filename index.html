Guía de Tecnologías Web y Java EE

Esta página informativa reúne los conceptos y tecnologías clave para el desarrollo de aplicaciones web modernas, abarcando desde los fundamentos de HTML y CSS hasta herramientas avanzadas de Java EE como JPA, Hibernate, EJB y el servidor GlassFish.
Explora las secciones para comprender cómo se relacionan y aplican en proyectos reales

Tecnologías Web: Frontend

  • HTML: Lenguaje de marcado que define la estructura básica de las páginas web. Permite organizar el contenido en encabezados, párrafos, listas, tablas, formularios y más.
  • CSS: Lenguaje de estilos utilizado para definir la apariencia visual de las páginas HTML. Permite aplicar colores, fuentes, márgenes, layouts y adaptabilidad a dispositivos.
  • Bootstrap: Framework CSS que facilita el diseño responsivo y componentes visuales modernos como menús, tarjetas, formularios y grids, acelerando el desarrollo frontend.

Tecnologías Java EE: Backend y Presentación

  • Servlet: Componentes Java que procesan peticiones HTTP y generan respuestas dinámicas. Son la base de muchas aplicaciones web Java.
  • JSP (JavaServer Pages): Páginas que combinan HTML con código Java para generar contenido dinámico en el servidor, facilitando la creación de interfaces web.
  • JSF (JavaServer Faces): Framework Java para construir interfaces de usuario basadas en componentes reutilizables, gestionando el ciclo de vida de la vista y la navegación.
  • EJB (Enterprise JavaBeans): Componentes de servidor que encapsulan la lógica de negocio, transacciones y seguridad en aplicaciones empresariales Java.

Persistencia de Datos y ORM

  • JNDI (Java Naming and Directory Interface): API de Java que permite buscar y acceder a recursos como bases de datos, servicios y objetos remotos en un entorno de servidor.
  • Persistencia: Concepto que se refiere al almacenamiento y recuperación de datos de manera permanente, generalmente en bases de datos.
  • JPA (Java Persistence API): Estándar de Java para el mapeo objeto-relacional (ORM), facilitando la gestión de entidades y operaciones CRUD sobre bases de datos.
  • Hibernate (ORM): Framework de ORM para Java que implementa JPA y simplifica la interacción entre objetos Java y tablas de bases de datos relacionales.
  • DAO (Data Access Object): Patrón de diseño que separa la lógica de acceso a datos de la lógica de negocio, facilitando el mantenimiento y pruebas del código.

Herramientas y Modelos de Desarrollo

  • Maven: Herramienta de gestión de proyectos Java que automatiza la compilación, pruebas y dependencias.
  • POM (Project Object Model): Archivo XML central de Maven donde se definen las dependencias, plugins y configuración del proyecto.
  • Model: En el contexto MVC, el "Model" representa la lógica de negocio y los datos de la aplicación, separando la presentación y el control.

Servidor y Despliegue

  • Servidor GlassFish: Servidor de aplicaciones Java EE que permite desplegar, administrar y ejecutar aplicaciones empresariales, soportando tecnologías como Servlets, JSP, EJB y JPA.
