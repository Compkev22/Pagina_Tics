Servidor y Despliegue

GlassFish Server y el proceso de despliegue en Java EE

¿Qué es GlassFish Server?

GlassFish Server es un servidor de aplicaciones de código abierto compatible con la especificación Java EE. Proporciona el entorno de ejecución necesario para desplegar, administrar y ejecutar aplicaciones empresariales basadas en Java, como las que utilizan Servlets, JSP, EJB y JPA.

  • Servidor de Aplicaciones: Proporciona un contenedor web y un contenedor EJB.
  • Soporte Java EE: Implementa las APIs de Java EE, garantizando la compatibilidad con las tecnologías estándar.
  • Herramientas de Administración: Ofrece una consola de administración web para gestionar las aplicaciones y la configuración del servidor.

Tipos de Servidores de Aplicaciones

Existen diferentes tipos de servidores de aplicaciones:

  • Contenedores Web: Servidores ligeros que solo implementan el contenedor de Servlets y JSP. Ejemplo: Apache Tomcat. Ideales para aplicaciones web simples.
  • Servidores de Aplicaciones Completos: Implementan toda la especificación de Java EE, incluyendo EJB, JPA, JMS, etc. Ejemplos: GlassFish Server y WildFly. Perfectos para aplicaciones empresariales grandes.

Proceso de Despliegue (Paso a Paso)

  1. Empaquetado del Proyecto: Desde tu IDE, utiliza Maven para empaquetar tu proyecto en un archivo .war.
    mvn package
    El archivo mi-proyecto.war se genera en la carpeta target/.
  2. Acceso a la Consola de Administración: Abre tu navegador y navega a http://localhost:4848.
  3. Subida de la Aplicación: En la consola, ve a la sección "Applications" y haz clic en "Deploy". Sube tu archivo .war desde la carpeta target/.
  4. Verificación del Despliegue: Una vez completado el despliegue, GlassFish indicará que la aplicación se ha subido correctamente. Puedes acceder a ella en http://localhost:8080/nombre-del-proyecto/.

Configuración del Servidor

El servidor no solo ejecuta la aplicación, sino que también gestiona los recursos que esta utiliza. Un ejemplo clave es la conexión a la base de datos:

  • DataSources: Un DataSource es un objeto que el servidor utiliza para gestionar un pool de conexiones a una base de datos. Se configura una vez en la consola de administración de GlassFish y la aplicación puede acceder a él a través de JNDI, evitando tener que configurar la conexión en cada clase.
Ejemplo de acceso a DataSource vía JNDI: 
InitialContext ctx = new InitialContext();
DataSource ds = (DataSource) ctx.lookup("java:comp/env/jdbc/miDataSource");
Connection conn = ds.getConnection();

Diferencia entre Archivos .war y .ear

  • .war (Web Application Archive): Se usa para empaquetar aplicaciones web que consisten en Servlets, JSP, JSF, HTML, CSS y JavaScript. Es el formato más común para proyectos web.
  • .ear (Enterprise Application Archive): Se usa para empaquetar aplicaciones empresariales más grandes y complejas que contienen múltiples módulos, como .war y .jar (que pueden contener EJB).

Aplicaciones y Ventajas

  • Despliegue de aplicaciones web y empresariales de mediana a gran escala que utilizan tecnologías Java EE.
  • Proporciona un entorno de ejecución completo y robusto, simplifica el despliegue de archivos .war y .ear, y ofrece herramientas de administración para monitorear la aplicación.

Conclusión

El servidor de aplicaciones GlassFish es un pilar central en el despliegue de aplicaciones Java EE. Proporciona un entorno robusto y completo que permite ejecutar componentes empresariales, garantizando seguridad, escalabilidad y un ciclo de vida bien definido para la aplicación.
