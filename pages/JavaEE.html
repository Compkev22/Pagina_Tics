Tecnologías Java EE – Backend y Presentación

Servlet, JSP, JSF, EJB

¿Qué es un Servlet?

Los Servlets son componentes de Java que se ejecutan en un servidor web. Su función es procesar peticiones de clientes (típicamente HTTP) y generar respuestas dinámicas. Son la base de muchas aplicaciones web en Java, actuando como el "controlador" que recibe las peticiones, las procesa, y genera una respuesta. 

import java.io.IOException;
import java.io.PrintWriter;
import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
 
@WebServlet("/hola")
public class HolaServlet extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response)
    throws ServletException, IOException {
        response.setContentType("text/html");
        PrintWriter out = response.getWriter();
        out.println("
Hola Mundo desde un Servlet!
");
    }
}

¿Qué es un JSP (JavaServer Pages)?

JSP es una tecnología que permite incrustar código Java directamente en páginas HTML. Esto facilita la creación de contenido dinámico de manera más sencilla que con los Servlets puros, ya que el desarrollador puede centrarse en la estructura de la interfaz web con la potencia de Java. El servidor compila los archivos JSP en Servlets, lo que permite un renderizado eficiente. 

<%@ page language="java" contentType="text/html; charset=UTF-8" pageEncoding="UTF-8"%>
<!DOCTYPE html>
<html>
<head>
    <title>Página JSP</title>
</head>
<body>
    <h1>Hola Mundo desde una página JSP!</h1>
    <p>La fecha actual es: <%= new java.util.Date() %></p>
</body>
</html>

¿Qué es JSF (JavaServer Faces)?

JSF es un framework para construir interfaces de usuario web basadas en componentes reutilizables. A diferencia de JSP, JSF utiliza un modelo de componentes y un ciclo de vida de la petición para gestionar la interacción del usuario, la validación de datos y la navegación entre páginas. Se enfoca en el desarrollo de aplicaciones web ricas en funcionalidad y eventos. 

<html xmlns="http://www.w3.org/1999/xhtml"
      xmlns:h="http://xmlns.jcp.org/jsf/html">
  <h:head>
    <title>Página JSF</title>
  </h:head>
  <h:body>
    <h:form>
      <h:outputLabel for="nombre" value="Introduce tu nombre:" />
      <h:inputText id="nombre" value="#{saludoBean.nombre}" />
      <h:commandButton value="Saludar" action="saludo" />
    </h:form>
  </h:body>
</html>

¿Qué son EJB (Enterprise JavaBeans)?

EJB son componentes de servidor que encapsulan la lógica de negocio en aplicaciones empresariales Java. Ofrecen servicios avanzados como gestión de transacciones, seguridad, persistencia de datos y concurrencia. EJB facilita el desarrollo de aplicaciones distribuidas y escalables, gestionando automáticamente aspectos complejos de la infraestructura. 

@Stateless
public class UsuarioService {
    @PersistenceContext(unitName="mi-persistencia")
    private EntityManager em;

    public void crearUsuario(Usuario usuario) {
        em.persist(usuario);
    }

    public Usuario obtenerUsuario(long id) {
        return em.find(Usuario.class, id);
    }
}

Aplicaciones y Ventajas

  • Aplicaciones: Desarrollo de aplicaciones web dinámicas, portales empresariales y sistemas de gestión complejos.
  • Ventajas: Permite crear páginas web dinámicas (Servlets/JSP), ofrece un modelo de componentes para la UI (JSF) y proporciona servicios empresariales robustos (EJB), facilitando un desarrollo estructurado y escalable.

Conclusión

Las tecnologías Java EE (Servlets, JSP, JSF y EJB) son la base para construir aplicaciones web robustas y escalables. Su uso permite separar las responsabilidades del software y facilita un desarrollo estructurado y mantenible, lo que es esencial para proyectos empresariales.
